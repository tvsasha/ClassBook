# ClassBook

**ClassBook** - веб-приложение электронного журнала для частной школы. Проект объединяет расписание, оценки, посещаемость, домашние задания, итоговые оценки и управленческую аналитику в одном аккуратном интерфейсе.

Система разрабатывается как дипломный проект: с реальными ролями, разграничением доступа, адаптивным интерфейсом и возможностью развернуть приложение локально или на сервере через Docker.

## Что умеет

- Вести электронный журнал по классам, предметам и урокам.
- Ставить оценки и отмечать посещаемость.
- Заполнять темы уроков и домашние задания.
- Смотреть дневник ученика и данные ребенка в кабинете родителя.
- Управлять пользователями, ролями, классами и предметами.
- Работать с расписанием в недельной сетке.
- Формировать итоговые оценки по периодам.
- Показывать директору сводную аналитику по учебному процессу.
- Поддерживать светлую и темную тему.
- Работать с телефона, планшета и компьютера.

## Роли

| Роль | Возможности |
| --- | --- |
| Администратор | Пользователи, роли, классы, предметы, доступы, QR-вход, настройки данных |
| Менеджер расписания | Недельная сетка расписания, переносы, копирование уроков и недель |
| Учитель | Мои уроки, журнал, оценки, посещаемость, темы, домашние задания |
| Директор | Просмотр журналов, расписания и аналитики без ежедневного ручного ввода |
| Родитель | Успеваемость ребенка, посещаемость, домашние задания, итоговые оценки |
| Ученик | Расписание, оценки, домашние задания, посещаемость, итоговые оценки |

## Технологии

- **Backend:** ASP.NET Core / .NET 10
- **Frontend:** React 19 + Vite
- **Database:** Microsoft SQL Server
- **ORM:** Entity Framework Core
- **Auth:** cookie-based authentication, роли и серверные проверки доступа
- **Deployment:** Docker, Docker Compose
- **Дополнительно:** QR-доступ, аудит действий, экспорт итоговых оценок

## Архитектура проекта

```text
ClassBook/
+-- Application/          # бизнес-логика, фасады, DTO
+-- Controllers/          # API-контроллеры ASP.NET Core
+-- Domain/               # доменные сущности
+-- Infrastructure/       # DbContext и инфраструктурный слой
+-- Migrations/           # миграции EF Core
+-- ClientApp/            # React-приложение
+-- wwwroot/              # собранный фронтенд и статические файлы
+-- scripts/              # SQL-скрипты обслуживания данных
+-- Dockerfile
+-- docker-compose.yml
```

## Быстрый запуск через Docker

1. Создайте файл окружения:

```bash
cp .env.example .env
```

2. В `.env` задайте надежный пароль SQL Server:

```env
MSSQL_SA_PASSWORD=Change_me_to_a_strong_password_2026!
```

3. Запустите проект:

```bash
docker compose up -d --build
```

4. Откройте приложение:

```text
http://localhost:8080/app/
```

При первом запуске приложение применит миграции базы данных автоматически.

## Локальная разработка

Backend:

```bash
dotnet restore
dotnet run
```

Frontend:

```bash
cd ClientApp
npm install
npm run dev
```

Сборка фронтенда:

```bash
cd ClientApp
npm run build
```

## Безопасность

В репозиторий нельзя добавлять:

- реальные пароли сервера или базы данных;
- `.env` с боевыми значениями;
- резервные копии базы данных;
- персональные данные учеников и родителей;
- закрытые ключи, токены и cookie.

Для примеров используются только безопасные шаблоны из `.env.example`.

## Интерфейс

ClassBook ориентирован на ежедневную работу школы: интерфейс старается быть спокойным, понятным и информативным без перегруженности. Основной упор сделан на быстрый доступ к журналу, расписанию, оценкам, посещаемости и отчетам.

## Статус

Проект активно развивается в рамках дипломной работы. Основные направления развития:

- улучшение аналитики директора;
- расширение отчетов и выгрузок;
- сценарий завершения учебного года;
- дополнительная полировка мобильной версии;
- более строгие политики безопасности для промышленного развертывания.

## Назначение

ClassBook показывает, как может выглядеть современная школьная информационная система для частной школы: с понятными кабинетами для всех участников учебного процесса, актуальными данными и аккуратной архитектурой.
